As of this time, ensemble applications seem to only work on Chrome. Apologies to Firefox and IE users!

This tutorial will walk you through the process of using ensemble to make a sample game. The completed source code for the game we'll be making, Lovers and Rivals, is packaged with ensemble in the sampleGame folder. However, sometimes the best way to learn is to build something yourself from scratch.

So, let's start at the very beginning! Before you actually do any coding, you'll want to set up a localhost. Once that's out of the way, it all starts with an html file, which we'll call loversAndRivals.html. You'll want to fill it up with your typical html friends, the html tag, the body tag, the title tag, you know the drill.

Once you have the basics in place, you'll want to add the following line inside your head tag: 

	<script src="ensemble.js"> </script>

And that should get you access to all of the wonders of ensemble! Simple as that!

You should also include the following lines: 

<script>
	document.addEventListener('ensembleLoaded', function (e) {
		var loadResult = ensemble.init();
		console.log(loadResult);
	}, false);
</script>

This ensures that your page won't start up until all of ensemble has been successfully loaded. To double check that you have everything up and running, open up the Chrome developer tools, and verify that a success message was printed to the console. Congratulations, you're ready to rock and roll!

We'll start making some game specific code in the next tutorial. For now, go down this sanity-check-list!

  • You're using Chrome as your browser
  • You've set up a localhost as outlined here.
  • Your html file that you're working in looks a little something like this: 
    
<html>
<head>
	<title>My Fun Game!</title>
	<script src="ensemble.js"></script>
</head>
<body>
	<script type="text/javascript">

		document.addEventListener('ensembleLoaded', function (e) {
			//Init ensemble
			var loadResult = ensemble.init();
			console.log(loadResult);
		}, false);
	</script>
</body>
</html>

Once you've gotten all of that in place, you're ready to move on to Part 2!
